Cloud storage services facilitate efficient data sharing among groups, but the inherent distrust of cloud servers necessitates robust remote data possession checking (RDPC) protocols to ensure data integrity. Traditional RDPC methods often rely on public key infrastructure (PKI), which suffers from security flaws and cumbersome certificate management. In contrast, identity-based cryptography (IBC) offers a solution but is hindered by key escrow issues. To address these challenges, we propose a new RDPC protocol utilizing certificate less signatures, where each user's private key combines a partial key from a group manager with a self-chosen secret. This approach associates public keys with unique user identities, eliminates the need for certificates, supports public verification without data downloads, and allows for efficient user revocation, all while ensuring security based on the assumptions of computational Diffie-Hellman (CDH) and discrete logarithm (DL).
